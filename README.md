# Social Spark App ✨

## Student Information
- Module: Introduction to Mobile Application Development
- Assignment: Assignment 1
- App Name: Social Spark

## Overview
Social Spark is a native Android app developed in Kotlin using Android Studio. The purpose of the app is to help users maintain social connections during busy days by suggesting simple social actions based on the time of day.

## Problem Background
People often become disconnected during busy schedules. This app provides small social “sparks” that encourage users to interact with friends or family depending on whether it is morning, afternoon, or night.

## Objective
The objective of this app is to:
- accept user input for the time of day,
- use conditional logic to determine a matching suggestion,
- display the suggestion clearly,
- allow the user to reset the app,
- handle invalid input with a constructive error message.

## Features
- Text input for time of day
- Suggestion output based on user input
- Reset button to clear input and output
- Error handling for incorrect entries
- Logging for testing and understanding code flow
- Simple and engaging user interface

## Technologies Used
- Kotlin
- Android Studio
- ConstraintLayout
- GitHub
- GitHub Actions

## How the App Works
1. The user types either:
   - Morning
   - Afternoon
   - Night
2. The app checks the input using conditional logic.
3. A matching social suggestion is displayed.
4. If the input is incorrect, the app shows an error message.
5. The Reset button clears the entered text and result.

## Design Considerations
The app was designed to be:
- simple to use,
- visually clean,
- beginner-friendly,
- engaging through colour and emoji use,
- easy to test and maintain.

## Testing
The app was tested manually using an Android emulator in Android Studio.
The following was tested:
- valid input: Morning
- valid input: Afternoon
- valid input: Night
- invalid input: other words
- Reset button functionality

## GitHub Actions
GitHub Actions was added to automatically build the project whenever code is pushed to the repository. This helps ensure that the app can build successfully outside the local machine.
